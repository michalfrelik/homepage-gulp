CSS: Prefixy

Przeglądarki implementują różne feature-y w różnym czasie (i często każda po swojemu)

Prefixy pochodzą z historii przeglądarek. kiedyś trzeba było dopisywać prefix przy własności osobno dla każdej przeglądarki (starsze wersje dalej potrzebują) np. flexbox

  • display: -webkit-box (google chrome, safari)
  • display: -moz-box (firefox mozilla)
  • display: -ms-box (microsoft - internet explorer, edge)

Sprawdzenie do jakiej własności trzeba dopisywać prefixy i jakie dla jakich przeglądarek na stronie - CanIUse

Autoprefixer

Google fonts

Google fonts - serwis który pozwala nam na dołączenie zewnętrzych fontów do naszego projektu

zażółć gęślą jaźń

te 3 słowa zawierają wszystkie polskie znaki diakrytyczne

Latin Extended - trzeba zaznaczyc, żeby zaimportować polskie znaki

2 sposoby importu:

  1. Kopia arkusza stylów (link generowany podczas wyboru fontów) - wrzucamy w head, tam gdzie nasze style
  2. @import (polecane) - kopiujemy link @import (link generowany podczas wyboru fontów) i wklejamy przy naszych @importach

font-family: 'nazwa fontu' (podajemy w apostrofach, bo mogą zawierać spację)

Pliki z fontami mogą "ważyć", dlatego nie nadużywamy tego - może wpływać na czas ładowania strony
z drugiej strony fonty google mogą być zapisywane w pamięci podręcznej przeglądarki co powoduje szybsze ładowanie stron kolejnym razem
Warto ograniczyć się do 2 rodzje fontów, lepiej działa i ładniej wygląda (unikamy tzw. choinki)

GIT: branche

Branche (gałęzie)

  • pozwalają na pracę w wielu osiach; równolegle
  • pozwalają na bezkarne testowanie (możemy psuć dowolu i powrócić do wcześniejszej wersji lub możemy porównywać wersje)
  • służą do bardziej efektywnego rozwijania oprogramowania, pozwalją na pracę równoległą (kilka osób może pracować rónolegle nad jakąś funkcją i później możemy połączyć to w całość
  • pozwalają zrównoleglić pracę
  • zwiększają czytelność postępu prac nad projektem
  • ułatwiają wypuszczanie rzeczy na świat (wersja stabilna jest wypuszczona, wersja rozwijana na innym branchu

master - główny branch, domyślny

git branch - wypisuje nam branche, które mamy aktualnie w projekcie

git branch nazwaBrancha - tworzenie nowego brancha

git checkout nazwaBrancha - przełączenie na innego brancha

git push --set-upstream origin nazwaBrancha - polecenie pushujące nowego brancha na gh (terminal podpowiadą tą komendę)

git merge nazwaBrancha - ("merge-owanie" - łączenie branchów) wchodzimy na brancha, który chcemy, żeby został (będziemy dalej na nim pracować) i wpisujemy komendę, który branch ma zostać połączony

JS: Tablice

Tablica [array] != tabela (table)

Jak wygląda tablica?

na rysunku: tablica stringów i tablica obiektów

Tablica może przyjmować różne rzeczy. Otwieramy za pomocą nawiasu [, wypusujemy po przecinku i zamykamy]
index [element 0, element 1, element 2] == index [0, 1, 2] - indexy liczymy od zera

tablica[index] - pobieranie elementu z tablicy na danym miejscu

tablica.length - ile mamy ekementów (liczy od 1, nie od 0!)

Stringi zachowują się podobnie jak tablice np.
const name = "maciek"
console.log(name[1]) - a

Można też sprawdzić jak długi jest string np.
console.log(name.length) - 6 (liczy również spację jako znak)

Jak pobrać znnak na danym miejscu?
const test="co ten frontend"
test[0] - c
test[1] - o
test[2] - _
test.length - 15 (liczy spacje)

Jak dodać element do tablicy na końcu?
tablica.push(element) - można umieszczać "stringi", {obiekty}

Jak dodać element do tablicy na początku?
tablica.unshift(element)

Jak usunąć element z tablicy z początku?
tablica.shift(element)

Jak usunąć element z tablicy z końca?
tablica.pop(element)

Jak sprawdzić czy mamy element w tablicy?
tablica.indexOf(element)
jezeli otrzymamy wynik -1 ti znaczy, że nie ma takiego elementu

Jak usunąć n elementów od danej pozycji?
tablica.splice(index, liczbaElementów)

Jak pobrać n elementów od danej pozycji?
tablica.slice(index, stop) - pobiera ale nie usówa z tablicy

splice != slice

JS: Pętle

Po co nam pętle (loops)?

Powstały, żeby uprościć kod i nie duplikować zbędnych rzeczy. Wykorzystywane, gdy mamy w kodzie powtarzające sie rzeczy.

FOR
for (inicjalizacja; warunek; wyrażenie inkrementacji){ kod }

i - licznik pętli, wskazuje nam ile racy coś się wykonało (używamy kolejnych liter alfabrtu i,j,k,..)

Jak wygląda pętla for?
for (let i=0; i"<"3; i++){
conosle.log(`${i}wykonanie pętli`)}

Postinkremantacja i++ - dodaje wartość po wykonaniu kodu

Postdekrementacja i-- - odejmuje wartość po wykonaniu kodu

Preinkremantacja ++i - dodaje wartość przed wykonaniem kodu

Predekrementacja --i - odejmuje wartość przed wykonaniem kodu

Pętli możemy użyć do wypisywania elementów
const elements=['a', 'b', 'c'];
for(let i=0; i '<' elements.length; i++){
console.log(`na miejscu ${i} jest ${elements[i]}`}

WHILE
while (warunek logiczny){ kod } - trzeba uważać, żeby nie zawiesić komputera, może się tak stać jak zrobimy pętlę nieskończoną

Jak wygląda pętla whlie?
let n=9;
do {
console.log(n);
n = n + 3;
} while (n'<'9);
console.log(n)

DO WHILE
do { kod } while (warunek logiczny) - pierw wykonuje kod, potem sprawdza warunek logiczny, zawsze się wykona

Jak wygląda pętla do ... whlie?
let n=3;
while (n'<'9){
console.log(n);
n = n + 3}
console.log(n)

FOR IN
for (własność in obiekt) - używamy do obiektów, głównie do wypisywania obiektu

Jak wygląda pętla for ... in?
const person = {
name: 'maciek'
age: 31
}
for (let propertyName in person){
console.log(`${propertyName}: ${person[propertyName]}`)
}

FOR OF
for (element of tablica) - wypisywanie elementów z tablic bez tworzenia pętli

Jak wygląda pętla for ... of?
const elements = ['a', 'b', 'c'];
for (let element of elements){
console.log(element);
}

JS: Przechowywanie danych

COOKIES - "twór" wykorzystywany do komunikacji między serwerem a klientem, używane między innymi do śledzenia użytkowników, zapisywania prostych rzeczy

document.cookie - pobieranie ciastka

document.cookie = "nazwa = wartość" - dodaje ciastka ciastka - COOKIE

Web storage dzieli się na local i session storage

localtorage - do ręcznego wyczyszczenia (trwalsze); dane istnieją tak długo, aż użytkownik je wyczyści

sessiontorage - do zamknięcia sesji (zakładki przeglądarki); dane istnieją do momentu zamknięcia sesji, odświerzenie strony

localStorage.setItem('nazwa', wartość) - działa podobnie jak obiekt js - dodaje element do local storage

const.jsonPerson = JSON.stringify(nazwa) - przetwarza obiekt w skompresowanego JSON-a w string, umożliwia pobranie obiektu do local storage

localStorage.getItem('nazwa') - pobiera element z local storage

const.newPerson = JSON.parse(nazwa) - przywraca normalny obiekt ze skompresowanego JSON-a (odwrotne do JSON.stringify)

stringify i parse - operację przetwarzania obiektu na skompresowany JSON nazywa się spłaszczaniem danych - spłaszczamy je po to, zeby można było je łatwo przechowywać

localStorage.removeItem('nazwa') - usuwa element z local storage

Frontendowiec vs projektowanie graficzne

Czy frontendowiec musi być grafikiem?
NIE

Zasady ułatwiające projektowanie

  1. Zacznij od prawdziwej treści.
  2. Naszkicuj wstępny układ na papierze.
  3. Porzuć wizję strony idealnej, zacznij od najniższego poziomu, który daje jakąś wartość.
  4. Działaj krok po kroku (one stap a time) - główna treść, nawigacja ...
  5. Trzymaj się siatki / grida.
  6. Zacznij od czarnego, białego i odcieni szarości.
    Nie zaczynaj od razu w kolorze. Zaczynamy od czarno-buialego, dzięki czemu skupimy się na treści, ułożeniu. Na końcu dopieramy kolory.
  7. Kolory zostaw na koniec i ogranicz ich liczbę do minimum - tekst czytelny, duzy kontrast.
  8. Nie wymyślaj koła na nowo. Ludzie lubią rzeczy, które znają.
  9. Wykorzystaj gotowe elementy / znajdź UI KIT (zestaw gotowych komponentów - figma).
  10. Kopiuj!
