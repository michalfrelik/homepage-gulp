Logika w JS

Operatory porównania

!=- nie równa się

if ('JavaScript' != 'Java') { //wykonaj kod jeśli }

Porównania:

> - lewa strona większa od prawej
< - prawa strona większa od lewej
>= - lewa strona większa równa od prawej
<= - prawa strona większa równa od lewej
== - obie strony równe
=== - obie strony równe (typeof - sprawdza typ liczba czy string)
!= - obie strony różne
!== - obie strony różne (typeof - sprawdza typ liczba czy string)

Zmienna typeof - definiuje typ danych, czy jest do liczba czy string

const liczba = 7 - liczba
const liczbaString = '7' - string (apostrof ' ' robi różnicę)

liczba == liczbaString - true- JS konwertuje wszystko na string, dlatego równanie jest prawdziwe
liczba === liczbaString - false- typeof sprawdza dokładniej typ danych, rozróżnia string od liczby

Zaleca się używanie potrójnego porównania (typeof) - kod będzie bezpieczniejszy
liczba != liczbaString - false
liczba !== liczbaString - true

Falsy - 6 wartości:

  • false
  • '' ; "" ; `` - pusty string (w JS można używać wszystkich apostrofów - zalecane '')
  • 0
  • null
  • undefined
  • NaN - not a number
  • false

Truthy - cała reszta:

  • 'tekst'
  • 12
  • -4
  • {}
  • []
  • true itd.

AND - && - wszystkie warunki muszą być true

if ((myVariable == 4) && (myVariable == 'tekst')) {...}

OR - || - przynajmniej jeden warunek ma być true

if ((myVariable == 4) || (myVariable != 'tekst')) {...}

NOT - ! - odwraca wartość wyrażenia logicznego

if (!('JavaScript' == 'Java') {...}
!false => true
!true => false

if ((myVariable == 4) {
//wykonaj kod jeśli true

}

JS : funkcje

Czym jest funkcja?

  • na wejściu dostaje dane - argumenty
  • dla każdego argumentu zwraca jakąś wartość
  • posiada określony wzór

f(x)=x*7
f(x) - argument
x*7 - wynik

function calculate (x) {return x*7;}
function - deklaracja
(x) - parametr
return - słowo kluczowe
x*7 - wynik

function calculate (x){
x=x+3;
console.log(x); retunr x*7:
}

W funkcji możemy zawierać wykonywanie akcji, czasami może być bez argumentu (funkcjie bezargumentowe - może tak być)

Wywołanie funckcji:

  • jej nazwa + argument
    calculate(2);
  • console.log(calculate(2))
  • const myCalculation = calculate (4)
    console.log(myCalculation)

Funkcja w JS:

  • Deklarujemy nazwę funckcji (nie zawsze trzeba)
  • W środku dokonujemy obliczenia / wykonujemy zadania
  • Funkcja przyjmuje argumenty (ale nie musi)
  • Funkcja zwraca zawsze coś (nawet jeśli nie mamy return - wtedy zwraca undefined)

Funkcje po co?

  • Deklarujemy kod, który się powtarza (DRY - Don't Repeat Yourself) - jeśli mamy coś (np. kod) co się powtarza, to dobrze to umieścić w funkcji
  • Zwiększamy czytelność kodu
  • Łatwiejsze szukanie błęów
  • Odpowiedzialność za wykonywanie zadania wjednym miejscu

Debugging- proces usuwania błędów i ich wyszukiwania

Co robić jak nie działa?

  1. Sprawdź czy plik jest zapisany.
  2. Sprawdź czy edytujesz właściwy plik
  3. Sprawdź czy odświerzyłeś przeglądarkę usuwając tzw. cache (oamięć tymczasowa)
    devtools -> Network -> Disable cache (usuwa pliki zapisane tymczasowo, powoduje wczytywanie wszystkich plików na nowo)
  4. Sprawdź czy nie masz błędów w edytorze / w devtoolsach
    • np. czerwona fala w edytorze
    • devtoolsy mówuą gdzie jest błąd, co jest nie tak
    • sprawdź DOM można wyłapać błędy w HTML-u
  5. Zajrzyj w zakładkę Network - widać czy coś się nie wczytuje
  6. "Pogadaj z kaczką" - przeczytaj niby komuś na głos
  7. Google it!
    • warto skopiować do googla nazwę błędu z konsoli
    • najlepiej odpowiedzi szukać do roku wstecz (częste zmiany we frontendzie)
      Nie korzystamy ze strony w3schools - dużo przeterminowanych informacji
      w3c != w3schools
  8. Stack Overflow - strona internetowa
    • warto patrzeć na odpowiedzi mające najwięcej plusów
    • najlepiej do roku wstecz

Normalizacja vs reset

Przeglądarka posiada swoje domyślne style - user agent stylesheet (devtoolsy)

Domyślne style nie wszędzie (nie w każdej przeglądarce) są takie same!

Reset - usówa domyślne style przeglądarek

Typowy reset :

*{
margin: 0;
padding: 0;
outline: none;
}

ul, ol {
list-style: none;
}

Nie stosujemy takiego resetu

Normalizacja - służy do wyrównywania styli w przeglądarkach, nie resetowniu

normalize.css - plik do pobrania

Ujednolica style między różnymi przeglądarkami

Nejlepsza praktyka:
normalize.css
+
html{
box-sizing: border-box;
}
*, *:before, *:after {
box-sizing: inherit;
}

Wszystkie elementy są domyślnie - border-box (nie trzeba definiować bodrer-boxów - wymiarry będą całkowite)
inherit - dziedziczenie

Zewnętrznych plików nie edytujemy - dobra praktyka

Flexbox

Flexbox - CSS

display: flex - zamienia domyślny display: block na nowy system pozycjonowania i rozmieszczania elementów - pracuje w dówch osiach (pionowa -column i pozioma-row)

flex-direction: (row - domyślne, column, row-reverse, column-reverse)

flex-wrap: (nowrap - domyślne, wrap, wrap-reverse) - zawijanie wierszy/kolumn

flex-flow: row nowrap; - domyślne (zastępuje dwie powyższe)

justyfi-content: (flex-start - domyślne(boxy przyklejają się do lewej/górnej krawędzi zależnie od flow-direction), flex-end, centerm, space-between(przerwa pomiędzy - dostosowuje do całej szerokości), space-around(dodaje jeszcze marginesy na końcach strony)) - ustawienie elementów wzdłuż osi flex-direction

align-content: (stretch - domyśnie (rozciąga maksymalnie), flex-start, flex-end, center, space-between, space-around) - ustawienie wierszy/kolumn wzdłuż osi przeciwnej do flex-direction (taki justyfi-content dla drugiej osi)

align-item: (stretch, flex-start, flex-end, center, baseline - elementy ustawiają się według wiersza tekstu w przypadku różnych wielkości fontów (teksty zaczynają się w tym samym miejcu)) - ustawienie elementów wzdłuż osi przeciwnej do flex-direction ale w obrębie pojedynczego wiersza)

align-self: (stretch, flex-start, flex-end, center, baseline) - ustawienie elementu wzdłuż osi przeciwnej do flex-direction (możemy pozycjonować pojedynczy element w wierszu)

order (0 , liczba całkowita (+/-)) ustala kolejność)

flex-grow: (0 , liczba/proporcja) - powoduje rośnięcie elementu (1 - domyślne)

flex-shtink: (1 , liczba/proporcja) - powoduje kurczenie elementu (1 - domyślne)

flex-grow i flex-shtink można stosować razem

flex-basis: (none, rozmiar bazowy) - domyślny rozmiar elementów

flex: flex-grow flex-shrink flex-basis - skrót
